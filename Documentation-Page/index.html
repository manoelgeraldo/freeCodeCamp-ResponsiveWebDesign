Introdução

JavaScript é uma linguagem de script de plataforma cruzada orientada a objetos. É uma linguagem pequena e leve. Dentro de um ambiente de host (por exemplo, um navegador da web), JavaScript pode ser conectado aos objetos de seu ambiente para fornecer controle programático sobre eles.

JavaScript contém uma biblioteca padrão de objetos, como Array, Date e Math, e um conjunto básico de elementos de linguagem, como operadores, estruturas de controle e instruções. O JavaScript básico pode ser estendido para uma variedade de propósitos, complementando-o com objetos adicionais; por exemplo:

  • O JavaScript do lado do cliente estende a linguagem central, fornecendo objetos para controlar um navegador e seu Document Object Model (DOM). Por exemplo, as extensões do lado do cliente permitem que um aplicativo coloque elementos em um formulário HTML e responda a eventos do usuário, como cliques do mouse, entrada de formulário e navegação de página.
  • O JavaScript do lado do servidor estende a linguagem central, fornecendo objetos relevantes para a execução de JavaScript em um servidor. Por exemplo, as extensões do lado do servidor permitem que um aplicativo se comunique com um banco de dados, forneça continuidade de informações de uma chamada para outra do aplicativo ou execute manipulações de arquivo em um servidor.

JavaScript e Java

JavaScript e Java são semelhantes em alguns aspectos, mas fundamentalmente diferentes em alguns outros. A linguagem JavaScript se assemelha a Java, mas não possui tipagem estática e verificação de tipo forte de Java. JavaScript segue a maioria da sintaxe de expressão Java, convenções de nomenclatura e fluxo de controle básico constrói que foi a razão pela qual foi renomeado de LiveScript para JavaScript.

Em contraste com o sistema de classes de tempo de compilação do Java construído por declarações, o JavaScript oferece suporte a um sistema de tempo de execução baseado em um pequeno número de tipos de dados que representam valores numéricos, booleanos e de string. JavaScript tem um modelo de objeto baseado em protótipo em vez do modelo de objeto baseado em classe mais comum. O modelo baseado em protótipo fornece herança dinâmica; ou seja, o que é herdado pode variar para objetos individuais. JavaScript também suporta funções sem quaisquer requisitos declarativos especiais. Funções podem ser propriedades de objetos, executando como métodos vagamente tipados.

JavaScript é uma linguagem de formato muito livre em comparação com Java. Você não precisa declarar todas as variáveis, classes e métodos. Você não precisa se preocupar se os métodos são públicos, privados ou protegidos e não precisa implementar interfaces. Variáveis, parâmetros e tipos de retorno de função não são digitados explicitamente.

Variáveis

Você usa variáveis ​​como nomes simbólicos para valores em seu aplicativo. Os nomes das variáveis, chamados identificadores, obedecem a certas regras.

Um identificador JavaScript deve começar com uma letra, sublinhado (_) ou cifrão ($); os caracteres subsequentes também podem ser dígitos (0-9). Porque JavaScript diferencia maiúsculas de minúsculas, as letras incluem os caracteres de "A" a "Z" (maiúsculas) e os caracteres "a" a "z" (minúsculas).

Você pode usar letras ISO 8859-1 ou Unicode, como å e ü nos identificadores. Você também pode usar as sequências de escape Unicode como caracteres em identificadores. Alguns exemplos de nomes legais são Number_hits, temp99 e _nome.

Declarando variáveis
Você pode declarar uma variável de três maneiras:

Com a palavra-chave var. Por exemplo, var x = 42. Esta sintaxe pode ser usado para declarar variáveis ​​locais e globais.

Simplesmente atribuindo a ele um valor. Por exemplo, x = 42. este sempre declara uma variável global. Ele gera um JavaScript restrito Atenção. Você não deve usar esta variante.

Com a palavra-chave let. Por exemplo, let y = 13. Esta sintaxe pode ser usado para declarar uma variável local de escopo de bloco. Veja o escopo da variável abaixo.

Escopo das variáveis

Quando você declara uma variável fora de qualquer função, ela é chamada de variável global, porque está disponível para qualquer outro código no documento atual. Quando você declara uma variável dentro de uma função, é chamada de variável local, porque está disponível apenas dentro daquele função.

JavaScript anterior ao ECMAScript 2015 não tem escopo de instrução de bloco; em vez disso, uma variável declarada dentro de um bloco é local para a função (ou escopo global) no qual o bloco reside. Por exemplo o seguinte código irá logar 5, porque o escopo de x é a função (ou global contexto) dentro do qual x é declarado, não o bloco, que neste caso é uma declaração if.

if (true) { var x = 5; } console.log(x); // 5

Este comportamento muda, ao usar a declaração let introduzida em ECMAScript 2015.

if (true) { let y = 5; } console.log(y); // ReferenceError: y is not defined
Variáveis globais

Variáveis ​​globais são na verdade propriedades do objeto global. Na web páginas, o objeto global é a janela, para que você possa definir e acessar variáveis ​​usando a sintaxe window.variable.

Consequentemente, você pode acessar variáveis ​​globais declaradas em uma janela ou quadro de outra janela ou quadro, especificando a janela ou quadro nome. Por exemplo, se uma variável chamada phoneNumber for declarada em um documento, você pode se referir a esta variável de um iframe como parent.phoneNumber.

Constantes

Você pode criar uma constante somente leitura, denominada constante com a palavra-chave const. o a sintaxe de um identificador constante é a mesma de uma variável identificador: deve começar com uma letra, sublinhado ou cifrão e pode conter caracteres alfabéticos, numéricos ou sublinhados.

const PI = 3.14;

Uma constante não pode mudar o valor por meio de atribuição ou ser declarada novamente enquanto o script está em execução. Ele deve ser inicializado com um valor.

As regras de escopo para constantes são as mesmas para permitir o escopo do bloco variáveis. Se a palavra-chave const for omitida, o identificador será considerado representam uma variável.

Você não pode declarar uma constante com o mesmo nome de uma função ou variável no mesmo escopo. Por exemplo:

// THIS WILL CAUSE AN ERROR function f() {}; const f = 5; // THIS WILL CAUSE AN ERROR ALSO function f() { const g = 5; var g; //statements } No entanto, os atributos do objeto não são protegidos, então a seguinte instrução é executado sem problemas. const MY_OBJECT = {"key": "value"}; MY_OBJECT.key = "otherValue";
Tipos de dados

O padrão ECMAScript mais recente define sete tipos de dados:

  • Seis tipos de dados que são primitivos:

    • Boolean. verdadeiro e falso.
    • Null. Uma palavra-chave especial denotando um valor nulo. Porque JavaScript faz distinção entre maiúsculas e minúsculas, nada não é o mesmo que Nulo, NULL ou qualquer outra variante.
    • Undefined. Uma propriedade de nível superior cujo valor é indefinido.
    • Number. 42 ou 3.14159.
    • String. "Olá"
    • Symbol (novo no ECMAScript 2015). Um tipo de dados cujas instâncias são único e imutável.
  • e Objetos
Embora esses tipos de dados sejam uma quantidade relativamente pequena, eles permitem que você para executar funções úteis com seus aplicativos. Objetos e funções são os outros elementos fundamentais da linguagem. Você pode pensar em objetos como recipientes nomeados para valores e funções como procedimentos que seu aplicativo pode executar.
Laço de repetição if e else
Use a instrução if para executar uma instrução se uma condição lógica for verdade. Use a cláusula opcional else para executar uma instrução se a condição é falso. Uma declaração if tem a seguinte aparência: if (condition) { statement_1; } else { statement_2; } condição pode ser qualquer expressão avaliada como verdadeira ou falsa. Vejo Booleano para uma explicação do que é avaliado como verdadeiro e falso. E se condição avaliada como verdadeira, instrução_1 é executada; de outra forma, instrução_2 é executada. declaração_1 e declaração_2 podem ser qualquer declaração, incluindo outras instruções if aninhadas.

Você também pode compor as instruções usando else se tiver vários condições testadas em sequência, como segue:

if (condition_1) { statement_1; } else if (condition_2) { statement_2; } else if (condition_n) { statement_n; } else { statement_last; } No caso de várias condições, apenas a primeira condição lógica que avaliado como verdadeiro será executado. Para executar várias instruções, agrupe -los dentro de uma instrução de bloco ({...}). Em geral, é uma boa prática sempre usar instruções de bloco, especialmente ao aninhar instruções if: if (condition) { statement_1_runs_if_condition_is_true; statement_2_runs_if_condition_is_true; } else { statement_3_runs_if_condition_is_false; statement_4_runs_if_condition_is_false; } É aconselhável não usar atribuições simples em uma expressão condicional, porque a atribuição pode ser confundida com igualdade ao olhar para cima o código. Por exemplo, não use o seguinte código: if (x = y) { /* statements here */ } Se você precisar usar um atribuição em uma expressão condicional, uma prática comum é colocar parênteses adicionais em torno da tarefa. Por exemplo: if ((x = y)) { /* statements here */ }
Laço de repetição while
Uma instrução while executa suas instruções enquanto uma condição especificada avalia como verdadeiro. Uma declaração while tem a seguinte aparência: while (condition) statement Se a condição se tornar falsa, instrução dentro do loop para a execução e o controle passa para o declaração seguindo o loop.

O teste de condição ocorre antes que a instrução no loop seja executada. E se a condição retorna verdadeira, a instrução é executada e a condição é testado novamente. Se a condição retornar falsa, a execução para e o controle é passado para a instrução seguinte ao while.

Para executar várias instruções, use uma instrução em bloco ({...}) para agrupar essas declarações.

Exemplo:

O seguinte loop while itera, desde que n seja menor que três:

var n = 0; var x = 0; while (n < 3) { n++; x += n; }

Com cada iteração, o loop incrementa n e adiciona esse valor a x. Portanto, x e n assumem os seguintes valores:

  • Após a primeira passagem: n = 1 and x = 1
  • Após a segunda passagem: n = 2 and x = 3
  • Após a terceira passagem: n = 3 and x = 6

Depois de concluir a terceira passagem, a condição n & lt; 3 não é mais verdadeiro, então o loop termina.

Declarando funções
Uma definição de função (também chamada de declaração de função ou função declaração) consiste na palavra-chave da função, seguida por:
  • O nome das Funções.
  • Uma lista de argumentos para a função, colocados entre parênteses e separados por vírgulas.
  • As instruções JavaScript que definem a função, entre curvas colchetes, {}.

Por exemplo, o código a seguir define uma função simples chamada square:

function square(number) { return number * number; }

O quadrado da função leva um argumento, chamado número. A função consiste em uma declaração que diz para retornar o argumento do função (ou seja, número) multiplicada por ela mesma. A declaração de retorno especifica o valor retornado pela função.

return number * number;

Parâmetros primitivos (como um número) são passados ​​para funções por valor; o valor é passado para a função, mas se a função mudar o valor do parâmetro, esta mudança não é refletida globalmente ou em a função de chamada.
